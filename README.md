# Coinplay Video Generator

Автогенерация коротких вертикальных **сюжетных** «brainrot»-мультиков (15–25 сек)
под Reels / Shorts / TikTok, которые **нативно продвигают бренд Coinplay**.

Та же модель работы, что у баннеров: **задал вертикаль/тему в Airtable → крон на
Railway всё сделал → готовый ролик прилетел в Telegram.** Но теперь это не «один
маскот читает закадр», а полноценный скит с **разными персонажами**, диалогами,
сюжетной аркой и брендовым пейоффом — как вирусные ролики, где, например, фрукты
спорят на прилавке.

```
Airtable(Pending)
  → GPT-4o пишет СЦЕНАРИЙ под вертикаль: идея + каст 2–4 персонажа + диалоги
      + сюжетная арка (хук → эскалация → твист → нативный бренд-пейофф → CTA)
  → Nano Banana Pro рисует РЕФЕРЕНС каждого персонажа (параллельно; опц. маскот)
  → Nano Banana Pro рисует КЕЙФРЕЙМ каждого шота, передавая референсы ВСЕХ
      персонажей в кадре (мульти-референс → герои on-model и в одной сцене)
  → Veo 3.1 оживляет каждый кейфрейм И ОЗВУЧИВАЕТ его: персонажи сами говорят
      свои реплики с липсинком + нативный эмбиент  (движок по умолчанию)
  → ffmpeg: склейка + хук-текст + лого + брендовая энд-карта
  → S3/R2 (опц. хостинг) + Telegram (доставка) → Airtable(Done)
```

Видео-движок переключается одной переменной `VIDEO_ENGINE` (см. §«Видео-движок»):
**`veo`** (по умолчанию) — персонажи сами говорят (ближе всего к референс-роликам);
**`kling`** — немой i2v + закадр ElevenLabs + пословные субтитры (дешевле, полный
контроль текста). Стоимость: Veo Fast ≈ **$3–5** за ролик, Kling ≈ **$2.5–3.5**.

---

## 0. Видео-движок: Veo 3.1 (персонажи говорят) vs Kling+TTS

Главный тумблер — переменная окружения **`VIDEO_ENGINE`**.

**`VIDEO_ENGINE=veo` (по умолчанию)** — каждый шот превращается в клип Veo 3.1
image-to-video с `generate_audio`, и модель **сама проговаривает реплики персонажей
с липсинком** + добавляет эмбиент. ElevenLabs/TTS не используется. Реплики из
сценария встраиваются в промпт Veo в его формате (`Dialogue:\nИмя: "реплика"`), язык
любой. Это то, что даёт «говорящих клубничек», как на референсе.

- Длину диктует сам клип (Veo — до 8 c на генерацию). Поэтому шотов меньше:
  `DEFAULT_SHOTS_VEO=3` → ~24 c. Сценарист в этом режиме пишет 1–2 коротких
  проговариваемых реплики на шот.
- Субтитров с караоке нет (нет TTS-таймкодов): `VEO_CAPTIONS=hook` показывает только
  крупный хук первые ~2 c, `off` — вообще без текста (как на референсе).
- Тиры качество↔цена (цена за секунду С аудио): `standard` $0.40, **`fast` $0.15**
  (дефолт), `lite` $0.05 (720p). Пример: 3 шота × 8 c на Fast ≈ **$3.6** + кейфреймы
  (~$0.3) + сценарий (~$0.01). На Lite те же 24 c ≈ **$1.6**.
- 720p и 1080p на standard/fast стоят одинаково → дефолт `VEO_RESOLUTION=1080p`.
- ⚠️ Все ролики Veo несут невидимый вотермарк Google **SynthID** (отключить нельзя).
- Тонкая настройка: `VEO_TIER`, `VEO_DURATION` (8s/6s/4s), `VEO_RESOLUTION`,
  `VEO_GENERATE_AUDIO`, `VEO_CAPTIONS`, `VEO_MUSIC_UNDER`, `VEO_SAFETY_TOLERANCE`,
  `DEFAULT_SHOTS_VEO`, `FAL_VEO_MODEL`.

**`VIDEO_ENGINE=kling`** — прежний маршрут: Kling i2v без звука, закадр ElevenLabs
(один голос + сдвиг тона на персонажа), пословные субтитры с раскраской по
говорящему. Дешевле, мультиязычность и точные слова/голос под полным контролем, но
**губы не совпадают с речью** (голос кладётся поверх). Здесь работают §6 (голоса),
`HERO_SHOTS`, `MUSIC_*`.

Что выбрать: нужен липсинк и вид как на референсе → `veo`. Нужно дёшево и точный
контроль текста/голоса/бренд-реплики, караоке-сабы → `kling`.

Совет: гонять идеи офлайн без трат — `python3 cli.py storyboard script.json` покажет
итоговый промт Veo с блоком диалога (то, что реально уйдёт в модель).

## 1. Что нового по сравнению с MVP

- **Veo 3.1 «персонажи говорят».** Движок по умолчанию: нативное аудио + липсинк,
  без TTS. Переключается на Kling+TTS одной переменной (см. §0).
- **Сюжет с разными персонажами.** Сценарист генерит каст (2–4 героя) с детальным
  визуальным «биографником» каждого, диалогами и сюжетной аркой. Раньше был один
  переиспользуемый маскот и сплошной закадр.
- **Идея под вертикаль.** Достаточно задать вертикаль (`casino`, `sportsbook`,
  `World Cup betting`, …) — модель сама придумает абсурдный скит. Можно задать и
  конкретный формат (см. ниже).
- **Голоса персонажей (режим kling).** Один голос ElevenLabs → разные тембры за счёт
  сдвига тона (длительность сохраняется, тайминги сабов валидны). Опционально можно дать
  отдельные `voice_id` на персонажа.
- **Нативная интеграция бренда.** Coinplay — это развязка/панчлайн истории, а не
  баннер «ЗАБЕРИ БОНУС». Опционально в финале появляется маскот-лис Coinplay.
- **Субтитры с раскраской по говорящему** — в спорах сразу видно, кто говорит.
- **Параллелизм** (референсы / кейфреймы+озвучки / i2v) — чтобы укладываться в
  бюджет крон-джоба.
- **Фолбэк при сбое i2v** — если видео-модель не отдала клип, шот собирается из
  кейфрейма эффектом Ken Burns, и ролик всё равно достраивается.
- **Починен баг со шрифтом** (см. §7): субтитры теперь реально рендерятся в
  брендовом Anton, а не молча падают в системный DejaVu.

## 2. Что проверено локально, а что требует ваших ключей

**Проверено реально (прогнано через ffmpeg), без внешних API:**
- весь компоновочный слой — нормализация шотов, склейка, мукс голос+музыка,
  прожиг субтитров, оверлей лого, энд-карта;
- пословные субтитры в **настоящем Anton** + **раскраска по говорящему**;
- сдвиг тона голоса **с сохранением длительности**, склейка озвучек, «немой бит»;
- видео-фолбэк Ken Burns из картинки;
- синтез промптов (референсы/кейфреймы/моушены) на готовом сценарии.

Смоук-тесты:
```bash
python3 _smoke_compose.py   # синтез-клипы → ffmpeg-сборка, синхрон, без обрезки
python3 _smoke_local.py     # НОВЫЕ куски: питч/тишина/конкат, Ken Burns,
                            # раскраска сабов, полная сборка с мульти-спикер сабами
python3 _smoke_veo.py       # Veo-режим: compose_native, сохранение нативного аудио,
                            # тихий фолбэк для клипов без звука, хук-субтитр
```

**Требует ваших ключей и одного live-смоука:** модули, дёргающие внешние API —
`script_writer.py` (OpenAI), `visuals.py` (fal), `voice.py` (ElevenLabs),
`clients.py` (Airtable/S3/Telegram). Код написан под актуальные на сейчас формы
этих API, но **имена эндпоинтов и полей fal меняются часто** — перед первым
прогоном сверьте `config.py → MODELS` и поля `payload` в `visuals.py` с актуальными
страницами моделей на https://fal.ai/models (особенно: для `nano-banana-pro/edit`
ключ мульти-референса может называться `image_urls`; у Kling длительность —
строка `"5"/"10"`; у Veo — целое число). Это единственное место, где может
потребоваться правка одной-двух строк.

## 3. Airtable: схема таблицы `Videos`

| Поле              | Тип             | Назначение                                              |
|-------------------|-----------------|---------------------------------------------------------|
| `Name`            | Single line     | Тема/бриф **или** вертикаль (обязательно)               |
| `Status`          | Single select   | `Pending` / `In progress` / `Done` / `Error`            |
| `Language`        | Single line     | `en`/`es`/`hr`/`lt`/`lv`/`ru`/`sr`… (дефолт `en`)        |
| `Vertical`        | Single line     | Опц. вертикаль (`casino`, `sportsbook`, …)              |
| `Format`          | Single line     | Опц. формат скита (см. §5)                               |
| `Duration`        | Number          | Опц. целевая длина, сек (15..25; дефолт 20)              |
| `Shots`           | Number          | Опц. число шотов (4..6; дефолт 5)                        |
| `Script_Override` | Long text       | Опц. готовый сценарий-JSON → пропускает LLM             |
| `Video_URL`       | URL             | Заполнится ссылкой (если включён S3/R2)                  |
| `Script`          | Long text       | Сгенерированный сценарий (для контроля)                 |
| `Error_Log`       | Long text       | Текст ошибки при сбое                                    |

Имена полей/таблицы переопределяемы через ENV (`AIRTABLE_*`), см. `.env.example`.

## 4. Схема сценария (то, что генерит/принимает система)

```jsonc
{
  "title": "Lemon vs Lime: The Payout",
  "concept": "коротко идея ролика",
  "format": "fruit-argument",
  "setting": "english description сцены/места",
  "cast": [
    {
      "id": "lemon",
      "name": "Lemon",
      "design": "ENGLISH визуальный биографник (для стабильного референса)",
      "personality": "loud, cocky",
      "voice": 1                       // целое 1..4 (маскот = 0); тембр персонажа
    }
  ],
  "shots": [
    {
      "characters": ["lemon", "lime"], // подмножество cast в кадре
      "visual": "ENGLISH описание кадра (что видим)",
      "motion": "ENGLISH движение камеры/персонажей",
      "dialogue": [                     // [] = «немой» комедийный бит
        {"speaker": "lemon", "line": "реплика на языке ролика"},
        {"speaker": "narrator", "line": "опц. закадровая строка"}
      ],
      "mood": "argue"
    }
  ],
  "on_screen_hook": "крупный текст-хук (язык ролика)",
  "brand_payoff": "нативная брендовая реплика-вывод (язык ролика)"
}
```

Правила, которые система **гарантирует сама** (нормализация в `script_writer.py`):
`design`/`visual`/`motion` — всегда по-английски (для генерации картинок/видео);
реплики/хук/пейофф — на языке ролика; у каждого персонажа уникальный `id` и
непротиворечивый `voice`; `characters` всегда подмножество `cast`; неизвестный
спикер схлопывается в `narrator`. Поддержан и **старый формат** (`shots[].narration`
без `cast`) — превратится в ролик «один рассказчик».

## 5. Форматы скитов (опционально)

Готовая библиотека «вайбов» (поле `Format`): `fruit-argument`, `courtroom`,
`sports-commentary`, `vs-duel`, `street-interview`, `monster-roommates`, `podcast`,
`infomercial`, `animals-meeting`. Если поле пустое — модель сама выберет формат
под вертикаль. Полный список и подсказки — в `brand/brand_prompts.py → FORMATS`.

---

## 6. Голос: один ключ ElevenLabs → разные персонажи

`VOICE_MODE=multi` (дефолт): профиль `0` — рассказчик и маскот; профили `1..4` —
персонажи. Если отдельные `ELEVEN_VOICE_1..4` не заданы, берётся дефолтный голос
языка + **сдвиг тона** (по умолчанию `+2.5 / −2.5 / +5 / −5` полутона). Сдвиг
сделан так, что **длительность не меняется**, поэтому пословные тайминги остаются
точными. `VOICE_MODE=single` — всё читает один голос. Подробности — `.env.example`.

---

## 7. Починка шрифта (важно)

В исходном проекте 3 из 4 `.ttf` в `brand/fonts/` оказались HTML-страницами
«404» (битые загрузки), включая дефолтный для субтитров **Anton**, а единственный
рабочий файл был внутренне помечен как тонкий ExtraLight. В итоге субтитры молча
падали в системный DejaVu, и брендовый шрифт не применялся.

Исправлено: подложены настоящие Anton / Oswald / Bebas Neue (из Google Fonts).
Чтобы баг не повторился, добавлен `fontutil.py` — бес-зависимостный парсер таблицы
имён шрифта: `config.py` теперь резолвит реальное имя семейства из файла, а не
доверяет имени файла. Если хотите свой шрифт — задайте `FONT_DISPLAY_PATH` (и при
желании `FONT_DISPLAY_NAME`).

## 8. Локальный CLI (дешёвая итерация над креативом)

```bash
# 1) Посмотреть, что уйдёт сценаристу (офлайн, без ключей):
python3 cli.py prompt --topic "World Cup BTTS" --vertical sportsbook \
        --format sports-commentary --lang hr

# 2) Сгенерить сценарий (нужен OPENAI_API_KEY) и сохранить:
python3 cli.py concept --topic "crypto casino" --vertical casino --lang en > script.json

# 3) Разложить готовый сценарий на ВСЕ промпты картинок/видео (офлайн):
python3 cli.py storyboard script.json

# 4) Собрать реальный mp4 из сценария (нужны FAL/ELEVENLABS/OPENAI):
python3 cli.py render script.json -o out.mp4 --lang en
```

`prompt` и `storyboard` работают **без секретов** — удобно крутить идеи и проверять
промпты, не тратя деньги на API. Пример готового сценария — `examples/sample_script.json`.

## 9. Запуск

**Локально:**
```bash
pip install -r requirements.txt        # ffmpeg должен быть установлен в системе
cp .env.example .env                    # заполнить ключи
python3 main.py                         # обработает Pending-записи (по бюджету)
```

**Railway:** деплой по `Dockerfile` (python:3.12-slim + ffmpeg + шрифты), запуск по
крону (`railway.toml`). За один запуск берётся `MAX_RECORDS_PER_RUN` записей (видео
долгое), остальное подхватит следующий крон. Бюджет — `MAX_JOB_SECONDS`.

## 10. Производительность и стоимость

- **Параллелизм** (`MAX_PARALLEL_JOBS`, дефолт 4): референсы каста параллельно;
  затем кейфреймы и озвучки параллельно; затем i2v параллельно. Сильно режет
  wall-clock и помогает уложиться в бюджет крона.
- **Hero-роут** (`HERO_SHOTS="0,4"`): отдельные шоты можно рендерить более дорогой/
  качественной видео-моделью (`MODELS.image_to_video_hero`). По умолчанию выкл.
- **Фолбэк** (`I2V_FALLBACK_KENBURNS=1`): сбой i2v → клип из кейфрейма (Ken Burns).
- **Стоимость** ≈ $2.5–3.5/ролик на Kling; основной драйвер — число шотов и
  включённый hero-роут. Картинки (референсы + кейфреймы) и озвучка — копейки на их фоне.

## 11. Структура

```
main.py                  оркестратор (Airtable → рендер → доставка), параллельные фазы
cli.py                   локальные команды: prompt / concept / storyboard / render
config.py                все настройки из ENV; резолв шрифта; профили голосов
fontutil.py              парсер имени семейства шрифта (защита от «битых» .ttf)
brand/
  brand_prompts.py       бренд-мир, библия маскота, форматы, промпт-сценарист
  fonts/                 Anton / Bebas Neue / Oswald (рабочие)
  assets/                лого, музыка
pipeline/
  script_writer.py       LLM-сценарий + жёсткая нормализация схемы (+back-compat)
  visuals.py             референсы каста, кейфреймы (мульти-референс), i2v + фолбэк
  voice.py               озвучка реплик, сдвиг тона, склейка, «немой бит»
  captions.py            ASS-субтитры с раскраской по говорящему
  compose.py             ffmpeg-сборка (нормализация/склейка/прожиг/энд-карта)
  media.py               ffmpeg-утилиты (тишина, питч, конкат, Ken Burns)
  falclient.py           очередь fal (картинки/видео)
  clients.py             Airtable / S3-R2 / Telegram
examples/sample_script.json   пример сценария для storyboard/render
_smoke_compose.py        офлайн-смоук сборки (синхрон/обрезка)
_smoke_local.py          офлайн-смоук новых кусков (голос/сабы/фолбэк/сборка)
```
